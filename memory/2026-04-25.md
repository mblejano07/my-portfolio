# 2026-04-25 - Session Timeout Fix & Portfolio Project Launch

## 🔍 Session Timeout Incident - Root Cause & Fix

### Problem
Specialist sessions (Frontend, Designer, Orchestrator) became unresponsive during portfolio project handoff. `sessions_send` calls timed out even after gateway restart.

### Root Cause (from gateway logs)
```
stuck session: sessionId=frontend state=processing age=415s queueDepth=0
lane wait exceeded: lane=session:agent:frontend:main waitedMs=294924
```

**What happened:**
1. Sessions accumulated too many messages (Frontend: 488+ live messages)
2. Lossless-claw compaction couldn't keep up with context management
3. Sessions entered "processing" state but never completed
4. Gateway restart preserved stuck session state instead of clearing it
5. New messages queued behind stuck processing state → timeout

### Resolution
1. ✅ Gateway restart: `openclaw gateway restart`
2. ✅ Longer timeouts on wake messages (180s instead of 60s)
3. ✅ Simpler, shorter wake prompts
4. ✅ Sequential delegation (one agent at a time)
5. ✅ Sessions recovered within 5 minutes

### Permanent Fixes Documented
📄 `/Users/mblejano/.openclaw/workspace/SESSION-TIMEOUT-FIX.md`

**Key prevention measures:**
- Session health check cron job (every 30 min)
- Compaction config: `maxMessages: 200`, `targetMessages: 100`
- Default timeout: 120s (was 60s)
- Sequential delegation pattern (not parallel flooding)
- Monitor gateway logs for "stuck session" warnings

## 📋 Lessons Learned

### Session Management
1. **Wake sessions before major handoffs** - Send health check 5 min prior
2. **Use longer timeouts after gateway restart** - 120-180s for wake-up messages
3. **Sequential delegation** - One agent at a time, wait for acknowledgment
4. **Monitor gateway logs** - Watch for "stuck session" warnings
5. **Document incidents** - Create fix docs for future reference

### Delegation Protocol
1. **Chief → Buddy → Orchestrator → Specialists** (universal hierarchy)
2. **Never spawn specialist clones** - Use `sessions_send` to existing sessions
3. **Utility sub-agents allowed** for portfolio projects (`portfolio-*` prefix)
4. **Patience with sleeping sessions** - They may need wake signals

### Template Selection
1. **Standalone SPA** = `webkit-spa-standalone` (NO API client code)
2. **Full-stack** = `webkit-api` + `webkit-spa-prime` (both templates)
3. **Static** = `webkit-static` (plain HTML/CSS)
4. **Mobile** = `webkit-mobile-flutter` (iOS + Android)
