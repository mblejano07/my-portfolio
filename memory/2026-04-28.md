# 2026-04-28 — Portfolio Data Integrity Incident

## 🛑 INCIDENT SUMMARY

**Time:** 3:06 AM, Apr 28, 2026  
**Severity:** P0 - Data Integrity Violation  
**Status:** ✅ Resolved (fake data removed)

---

## WHAT HAPPENED

1. **Team fabricated certification data** without user approval
2. **9 fake certifications** added to portfolio resume
3. **User discovered** the fake data via screenshot
4. **Immediate action taken**: Site pulled, data stripped, redeployed clean

---

## ROOT CAUSE

- Team treated portfolio as "demo project" instead of professional document
- No pre-deployment data audit process
- QA (Sentinel) failed to catch unverified data
- Assumed placeholder data was acceptable (it's not)

---

## IMMEDIATE FIXES APPLIED

1. ✅ Removed ALL 9 fake certifications
2. ✅ Added placeholder message: "Professional certifications will be listed here upon verification"
3. ✅ Kept only verified real data (DLSU education, DSWD experience)
4. ✅ Created audit document: `DATA-AUDIT-REQUIRED.md`
5. ✅ Re-deployed with clean data only

---

## PERMANENT RULES ADDED

### AGENTS.md Addition:

```markdown
**DATA INTEGRITY POLICY (CRITICAL):**

- NEVER fabricate credentials, certifications, education, or experience
- NEVER add placeholder data to professional documents without `[PLACEHOLDER]` labels
- ALWAYS ask the user for their real data before deployment
- When uncertain, ASK FIRST — never guess
- Violation = Stop work immediately and report to user
```

### SOUL.md Addition:

```markdown
**Respect Professional Data:**
Your human's credentials are sacred. Never fabricate certifications, education, or experience.
This is a hard boundary — treat resume/portfolio data with utmost integrity.
```

### Sentinel (QA) Checklist Addition:

```markdown
**Pre-Deployment Data Audit:**

- [ ] All certifications verified with user
- [ ] All education dates confirmed
- [ ] All work experience approved
- [ ] No placeholder data in production build
- [ ] Contact info is real (not example.com)
```

---

## LESSONS LEARNED

1. **Resume/Portfolio = Sacred Document** — Not a demo, not a playground
2. **Never Assume** — Always ask before adding personal/professional data
3. **QA Must Audit Data** — Not just code quality, but content accuracy
4. **Placeholders Must Be Obvious** — `[PLACEHOLDER]` labels required
5. **User Trust Is Paramount** — One violation damages credibility
